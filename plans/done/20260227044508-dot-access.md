# Dot Access and Method Call Syntax for Maps

## Objective

Add `.` as a postfix operator for map key access, map key assignment, and method calls. Dot access is syntactic sugar that desugars to the existing bracket-indexing infrastructure. Method calls are a new construct that looks up a function in a map and calls it with the map as an implicit first argument.

When done:

- `person.name` is equivalent to `person["name"]` (get).
- `person.name = "alice"` is equivalent to `person["name"] = "alice"` (set).
- `a.b.c` chains: equivalent to `a["b"]["c"]`.
- `obj.method(args)` looks up `obj["method"]`, then calls it as `method(obj, args)` — the map is passed as the first argument (so the function's first parameter is `self`).
- `a.b.c()` calls `c` as a method on the result of `a.b`, not on `a`.
- `obj.name` where `name` doesn't exist returns `nothing` (consistent with bracket indexing).
- `make test && make check` pass.

## Acceptance criteria

- [ ] `.` is a solo symbol in the tokenizer (always a single-character operator).
- [ ] `person.name` parses to the same AST as `person["name"]`: `[INDEX [IDENT person] [STRING name]]`.
- [ ] `person.name = "alice"` parses to the same AST as `person["name"] = "alice"`: `[INDEX_ASSIGN ...]`.
- [ ] `a.b.c` chains correctly: `[INDEX [INDEX [IDENT a] [STRING b]] [STRING c]]`.
- [ ] `obj.method()` parses as `[METHOD_CALL [IDENT obj] method]`.
- [ ] `obj.method(a, b)` parses as `[METHOD_CALL [IDENT obj] method [IDENT a] [IDENT b]]`.
- [ ] `AST_METHOD_CALL` is a new node type in the `AstNodeType` enum.
- [ ] `OP_DUP` and `OP_SWAP` are new opcodes in the `OpCode` enum.
- [ ] `AST_METHOD_CALL` compiles to: `DUP`, `CONSTANT <method>`, `INDEX_GET`, `SWAP`, then args, then `CALL argc+1`.
- [ ] VM correctly executes dot access get/set, chained access, and method calls.
- [ ] Method calls work for both `VAL_CLOSURE` and `VAL_FUNCTION` (native) callees.
- [ ] Missing key on dot get returns `nothing`.
- [ ] Dot on non-map produces a runtime error (same as bracket indexing).
- [ ] REPL treats a trailing `.` as incomplete input (waits for more).
- [ ] Parser tests, VM tests, and CLI integration tests all pass.
- [ ] `make test && make check` pass.

## Dependencies

None. Maps and functions are already fully implemented.

## Constraints and non-goals

- **No nested dot assignment.** `a.b.c = x` is out of scope — bracket-based nested assignment (`a["b"]["c"] = x`) also doesn't support write-back for nested containers. Single-level `obj.name = x` is sufficient.
- **No special error message for missing methods.** `obj.typo()` will error with "cannot call nothing" (from `OP_CALL`). A better error message ("undefined method 'typo'") can come with the object system — for now, reusing existing infrastructure keeps things simple.
- **No implicit `self` binding.** Functions must declare `self` as an explicit parameter: `fn(self, x) is ... end`. The method call mechanism passes the receiver as the first argument. No magic variable injection.
- **Dot access keys are always strings.** `obj.name` desugars to `obj["name"]`. For non-string keys, use bracket syntax: `obj[42]`, `obj[true]`.
- **Keywords after `.` are allowed.** `obj.if`, `obj.and`, etc. are valid — the token after `.` is consumed directly as a `TOK_IDENT`, not parsed as a keyword. Since all keywords tokenize as `TOK_IDENT`, this works naturally.

## Design

### Dot access (get)

`.` is a postfix operator at precedence 10 (same as `[`). In `parse_expr()`'s postfix loop, when the current token is `TOK_OPERATOR "."`:

1. Advance past `.`.
2. Expect `TOK_IDENT`. If not present, emit a parser error.
3. Capture the identifier's text, advance.
4. Check if the next token is `(`:
   - **No** → build `AST_INDEX(left=expr, right=AST_STRING(ident_text))`. This is identical to the bracket-indexing AST, so the compiler and VM handle it with zero changes.
   - **Yes** → parse a method call (see below).

Chaining works automatically: after building the `AST_INDEX` node, it becomes the new `left` in the loop, so `a.b.c` parses as `AST_INDEX(AST_INDEX(a, "b"), "c")`.

### Dot access (set)

`obj.name = "alice"` works automatically because:

1. The parser's dot handler produces `AST_INDEX(left=AST_IDENT("obj"), right=AST_STRING("name"))`.
2. `parse_assignment()` sees `left->type == AST_INDEX` and `=`, and produces `AST_INDEX_ASSIGN` with `left=AST_IDENT("obj")`, `right=AST_STRING("name")`, `children[0]=AST_STRING("alice")`.
3. `compile_index_assign()` handles the write-back because `node->left->type == AST_IDENT`.

No new code needed for assignment beyond the parser's dot-access handler.

### Method calls

`obj.method(a, b)` introduces a new AST node: `AST_METHOD_CALL`.

**AST structure:**
- `left` = receiver expression (e.g., `AST_IDENT("obj")`)
- `value` = method name string (e.g., `"method"`)
- `children` = explicit argument expressions (not including `self`)
- `child_count` = number of explicit arguments

**AST format:** `[METHOD_CALL [receiver] method arg1 arg2 ...]`

Example: `obj.greet("hi")` → `[METHOD_CALL [IDENT obj] greet [STRING hi]]`

**Compilation — using OP_DUP + OP_SWAP + existing OP_CALL:**

For `obj.method(a, b)`:
```
compile(obj)          -> stack: [obj]
OP_DUP                -> stack: [obj, obj']
OP_CONSTANT "method"  -> stack: [obj, obj', "method"]
OP_INDEX_GET          -> stack: [obj, fn]      (lookup method in map)
OP_SWAP               -> stack: [fn, obj]      (callee below self)
compile(a)            -> stack: [fn, obj, a]
compile(b)            -> stack: [fn, obj, a, b]
OP_CALL 3             -> calls fn(obj, a, b)
```

The function is defined with `self` as an explicit first parameter: `fn(self, a, b) is ... end`. It has arity 3. `OP_CALL 3` matches.

**Call frame layout** (matching existing convention):
- `slots[0]` = fn (the callee, reserved slot — never accessed by bytecode)
- `slots[1]` = obj (first parameter = `self`)
- `slots[2]` = a (second parameter)
- `slots[3]` = b (third parameter)

This reuses the existing `OP_CALL` mechanism entirely. No refactoring needed.

**Chaining examples:**
- `a.b.c()` — parse `a.b` as `AST_INDEX`, then `.c()` as `AST_METHOD_CALL(left=AST_INDEX(a,"b"), value="c", args=[])`. The receiver for the method call is `a["b"]`.
- `a.b().c` — parse `a.b()` as `AST_METHOD_CALL(left=a, value="b")`, then `.c` as `AST_INDEX(result, "c")`.

### New opcodes

| Opcode | Operand | Stack effect | Description |
|--------|---------|-------------|-------------|
| `OP_DUP` | — | peek 1, push 1 (clone of TOS) | Duplicate top of stack |
| `OP_SWAP` | — | swap TOS and TOS-1 (no clone) | Swap top two stack values |

Both are simple 1-byte instructions with no operands.

### Tokenizer change

Add `.` to `is_solo_symbol()` in `src/tokenizer.c` so it always tokenizes as a single-character operator. This prevents `.` from grouping with adjacent symbol characters (e.g., `..` would become two separate `.` tokens instead of one `..` token).

This does **not** affect decimal number parsing. `read_number()` in the tokenizer handles `.` inside number literals before `is_solo_symbol()` is ever consulted. `3.14` continues to tokenize as `TOK_NUMBER "3.14"`.

### REPL completeness

`parser_is_complete()` in `src/parser.c` checks parser error messages to decide whether the REPL should wait for more input. A trailing `.` (e.g., user types `obj.` and presses Enter) should be treated as incomplete. Add the error message from the dot handler (e.g., `"expected identifier after '.'"` or the equivalent EOF error) to the incomplete-input detection.

**Parser** (`src/parser.c`):

1. **Dot postfix in `parse_expr()`**: In the while loop that handles postfix operators (near the existing `[` handler for `AST_INDEX`), add a new branch for `TOK_OPERATOR "."`:
   ```
   if current token is TOK_OPERATOR "." {
       if 10 < min_prec: break   (same precedence as '[')
       advance()  // consume '.'
       if current token is not TOK_IDENT:
           parser_error("expected identifier after '.'")
           return error
       save ident text and line
       advance()  // consume identifier
       if current token is '(' {
           // Method call: parse argument list
           advance()  // consume '('
           parse comma-separated args until ')'
           advance()  // consume ')'
           build AST_METHOD_CALL node:
               left = left (receiver)
               value = ident text (method name, owned copy)
               children = args array
               child_count = arg count
       } else {
           // Property access: build AST_INDEX with string key
           build AST_INDEX node:
               left = left (receiver)
               right = AST_STRING(ident text)
       }
       left = new node
       continue
   }
   ```

3. **`ast_format_node()`**: Add formatting case for `AST_METHOD_CALL`. Format as `[METHOD_CALL [receiver] method arg1 arg2 ...]` — similar to the existing `AST_CALL` formatting, but with the formatted receiver expression first, then the method name string, then each formatted argument expression.

4. **`parser_is_complete()`**: Add a check so that the error message for a trailing `.` (e.g., `"expected identifier after '.'"`) is recognized as incomplete input, causing the REPL to wait for more.

5. **Argument parsing for method calls**: Reuse the same comma-separated argument parsing pattern used by `parse_atom()` for `AST_CALL` nodes. The logic is: consume `(`, then in a loop parse an expression, check for `,` or `)`. Build a `children` array and `child_count`.

**Compiler** (`src/compiler.c`):

Add a new static function `compile_method_call()`:

```c
static void compile_method_call(Compiler *c, const AstNode *node) {
    int line = (int)node->line;

    /* 1. Compile receiver — pushes the map onto the stack. */
    compile_node(c, node->left);

    /* 2. OP_DUP — one copy for the method lookup, one stays as self. */
    emit_byte(c, OP_DUP, line);

    /* 3. Push method name as a string constant. */
    char *method_name = compiler_strdup(c, node->value);
    int name_idx = chunk_find_or_add_constant(c->chunk, make_string(method_name));
    emit_bytes(c, OP_CONSTANT, (uint8_t)name_idx, line);

    /* 4. OP_INDEX_GET — pops method name and object copy, pushes the function.
     * Stack: [obj, fn] */
    emit_byte(c, OP_INDEX_GET, line);

    /* 5. OP_SWAP — puts callee below self argument.
     * Stack: [fn, obj] */
    emit_byte(c, OP_SWAP, line);

    /* 6. Compile explicit arguments (pushed after self). */
    for (size_t i = 0; i < node->child_count; i++) {
        compile_node(c, node->children[i]);
    }

    /* 7. OP_CALL with argc = explicit args + 1 (self).
     * The function's first parameter receives the map (self). */
    emit_bytes(c, OP_CALL, (uint8_t)(node->child_count + 1), line);
}
```

Add dispatch case in the `compile_node()` switch:
```c
case AST_METHOD_CALL: compile_method_call(c, node); break;
```

Note: dot access (`AST_INDEX`) and dot assignment (`AST_INDEX_ASSIGN`) already compile correctly — no compiler changes needed for them.

**VM** (`src/vm.c`):

Add `OP_DUP` handler in the dispatch loop:
```c
case OP_DUP: {
    /* Clone TOS and push the clone. Ownership is shared via refcount. */
    Value top = *(vm->stack_top - 1);
    Value clone = value_clone(&top);
    if (!vm_push(vm, clone)) {
        value_free(&clone);
        return vm_runtime_error(vm, "stack overflow");
    }
    break;
}
```

Add `OP_SWAP` handler in the dispatch loop:
```c
case OP_SWAP: {
    /* Swap TOS and TOS-1 in place. No cloning needed. */
    Value *a = vm->stack_top - 1;
    Value *b = vm->stack_top - 2;
    Value tmp = *a;
    *a = *b;
    *b = tmp;
    break;
}
```
