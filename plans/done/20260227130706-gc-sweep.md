# GC Task 4: Sweep Phase, GC Triggering, and String Interning

## Objective

Implement the sweep phase of mark-and-sweep garbage collection, enable automatic GC triggering based on allocation thresholds, and add string interning. After this task, the GC is fully operational: unreachable objects are reclaimed, the collector runs automatically, and identical strings share a single `ObjString` instance.

When done:

- `gc_sweep()` walks the object list, frees every unmarked object, and clears marks on survivors.
- `gc_collect()` runs `gc_mark_roots()` then `gc_sweep()`.
- GC triggers automatically in `gc_alloc()` when `bytes_allocated > next_gc`.
- `next_gc` grows after each collection (e.g., `next_gc = bytes_allocated * 2`).
- A string intern table deduplicates `ObjString` objects by content. `obj_string_new` returns the existing interned string if one matches.
- The sweep phase cleans dead strings from the intern table (weak references).
- `GC_LOG` compile-time flag prints collection stats (objects freed, bytes reclaimed).
- `make test && make check` pass.

## Acceptance criteria

- [ ] `gc_sweep()` in `src/gc.c` walks `gc.objects`, frees unmarked objects (type-dispatched content destructor + `free`), relinks the list around freed objects, clears `is_marked` on survivors.
- [ ] `gc_collect()` calls `gc_mark_roots()` then `gc_sweep()`.
- [ ] `gc_alloc()` checks `bytes_allocated > next_gc` and calls `gc_collect()` if true.
- [ ] `next_gc` is updated after each collection: `gc.next_gc = gc.bytes_allocated < 1024 ? 1024 : gc.bytes_allocated * 2`.
- [ ] String intern table: a hash table in `src/gc.c` mapping `uint32_t hash + content` to `ObjString*`.
- [ ] `obj_string_new()` checks the intern table first. If a matching string exists, returns the existing `ObjString*` (no new allocation).
- [ ] `obj_string_take()` checks the intern table first. If a match exists, frees the input buffer and returns the existing string.
- [ ] `gc_sweep()` removes dead (unmarked) strings from the intern table before freeing them.
- [ ] `value_clone()` for `VAL_STRING` no longer deep-copies: it returns the same `ObjString*` pointer (interned strings are shared).
- [ ] `value_free()` for `VAL_STRING` is a no-op for the `ObjString*` pointer (GC handles lifetime). Only nulls out the pointer.
- [ ] `value_equal()` for `VAL_STRING` uses pointer comparison (`a->string == b->string`) as the fast path, since interned strings with equal content are the same object.
- [ ] `GC_LOG` flag: when defined, `gc_collect()` prints to stderr: `"-- gc collected %zu objects, %zu bytes. %zu remaining.\n"`.
- [ ] Tests for sweep, automatic triggering, and string interning.
- [ ] `make test && make check` pass.

## Dependencies

- **gc-infrastructure** (task 1): Object list, `gc_alloc`, `gc_unlink`.
- **gc-objstring** (task 2): `ObjString` type on the GC list.
- **gc-mark** (task 3): `gc_mark_roots()`, `gc_mark_object()`.

## Constraints and non-goals

- **Refcounting still exists.** Refcount-based freeing coexists with GC sweep. Objects may be freed by either path. `gc_sweep` must handle objects that have already been freed by refcounting (they'll have been unlinked from the GC list by `gc_unlink`). Task 5 removes refcounting.
- **No generational collection.** Simple mark-and-sweep only. Generational or incremental GC is a future optimization.
- **Intern table is flat.** Simple open-addressing hash table. No need for a fancy data structure — string count is typically small in shell scripts.
- **COW still works.** Arrays and maps still use refcount-based COW. This is removed in task 5.

### Step 2: Implement gc_sweep

In `src/gc.c`, add static function `gc_sweep()`:

```
Walk gc.objects as a linked list (with prev pointer for relinking):
  For each object:
    if obj->is_marked:
      obj->is_marked = false   // clear for next cycle
      prev = obj
      move to next
    else:
      // Object is unreachable — free it.
      Unlink from list (prev->next = obj->next, or gc.objects = obj->next if first).
      Call free_object_contents(obj) to free internal allocations.
      Decrement gc.bytes_allocated.
      free(obj).
      Move to next (don't update prev).
```

The `free_object_contents()` helper (from task 1's `gc_free_all`) handles type-specific cleanup:
- `OBJ_FUNCTION`: Free `name`, `params` + strings, `chunk` (via `chunk_free` + `free`).
- `OBJ_CLOSURE`: Free upvalue pointer array (but NOT upvalues themselves — they're separate objects).
- `OBJ_ARRAY`: Free each element via `value_free`, free `data`.
- `OBJ_MAP`: Free each entry key+value via `value_free`, free `entries`.
- `OBJ_UPVALUE`: If closed, free `closed` value via `value_free`.
- `OBJ_STRING`: Free `chars`.

**Important**: When sweep frees an object, it must NOT call `gc_unlink` (which walks the list). Instead, it directly relinks the list as it iterates — this is O(n) total, not O(n^2).

**Important**: `value_free` called from `free_object_contents` may try to `gc_unlink` objects that are themselves being swept. To handle this, either: (a) make `gc_unlink` tolerate objects not on the list (no-op if not found), or (b) during sweep, skip `gc_unlink` calls by checking a `gc.sweeping` flag. Option (b) is cleaner — add `bool sweeping;` to the GC state, set it true during sweep, and have `gc_unlink` return immediately when sweeping.

Run `make test && make check`.

### Step 5: Implement the string intern table

In `src/gc.c`, add a string intern table:

**Data structure**: Open-addressing hash table of `ObjString*` pointers (weak references).

```c
#define INTERN_INIT_CAP 64
#define INTERN_MAX_LOAD 0.75

static ObjString **intern_table = NULL;
static size_t intern_cap = 0;
static size_t intern_count = 0;
```

**`intern_find(const char *chars, size_t length, uint32_t hash)`**: Walk the table looking for a matching string (same hash AND same chars via `memcmp`). Return the `ObjString*` if found, NULL otherwise.

**`intern_add(ObjString *str)`**: Insert into the table. Grow if load factor exceeded.

**`intern_remove(ObjString *str)`**: Remove a string from the table (used during sweep). Set the slot to a tombstone (or NULL + re-probe; tombstone is simpler).

**Update `obj_string_new()`**:
- Compute hash first.
- Call `intern_find(chars, length, hash)`. If found, return the existing ObjString (don't allocate).
- Otherwise, allocate via `gc_alloc`, copy chars, set fields, call `intern_add`, return.

**Update `obj_string_take()`**:
- Compute hash on the input buffer.
- Call `intern_find(chars, length, hash)`. If found, `free(chars)` (caller's buffer), return existing.
- Otherwise, allocate via `gc_alloc`, set `str->chars = chars` (take ownership), call `intern_add`, return.

**Update `gc_sweep()` (or add a pre-sweep step)**:
- Before the main sweep loop, walk the intern table and remove any entries whose `ObjString` has `is_marked == false`. This prevents the intern table from keeping dead strings alive.
- Alternatively, do this inline during sweep: when sweeping an `OBJ_STRING`, remove it from the intern table first.

Run `make test && make check`.

### Step 6: Update value_clone, value_free, and value_equal for interned strings

With interning, strings are shared objects managed by the GC. Value handling changes:

**`value_clone()` for `VAL_STRING`**:
- Simply copy the `ObjString*` pointer: `out->string = src->string;`. No new allocation, no `strdup`. The GC keeps the string alive as long as it's reachable.

**`value_free()` for `VAL_STRING`**:
- No-op: just set `v->string = NULL`. The GC handles freeing. Do NOT call `gc_unlink` or `free`.
- This is a significant change from the current behavior. Any code path that calls `value_free` on a string Value no longer frees memory — the GC sweep does.

**`value_equal()` for `VAL_STRING`**:
- Fast path: `a->string == b->string` → return true (pointer equality, O(1)).
- This is always correct because interning guarantees that identical content produces the same `ObjString*`.
- Remove the `strcmp` fallback (or keep it as a debug assertion).

**Important side effect**: Since `value_free` for strings is now a no-op, all the string Value lifecycle changes. Code that creates a temporary string Value and calls `value_free` to "release" it will no longer free memory. This is fine — the GC handles it — but means temporary strings may live longer than before (until the next collection). This is expected GC behavior.

**Update `obj_map_set()` and other functions that clone string keys**: These call `value_clone` which now just copies the pointer. This is correct — the GC keeps the string alive.

Run `make test && make check`.
