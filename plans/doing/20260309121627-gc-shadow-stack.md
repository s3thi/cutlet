# GC Shadow Stack (Replace gc_pin/gc_unpin)

## Objective

Replace the `gc_pin`/`gc_unpin`/`gc_pin_value` temporary root mechanism with a **shadow stack** that holds pointers to C local variables. Remove all existing pin/unpin calls from `src/vm.c` and delete the old API. After this task, the GC roots C locals by tracking their addresses, and the pinning API no longer exists.

## Background

When a VM opcode pops values off the VM stack into C local variables, those objects become invisible to the GC. If a subsequent allocation triggers `gc_collect`, the objects are swept. The current solution is `gc_pin(obj)` / `gc_unpin()`, which pushes the object's pointer value onto a fixed-size stack that `gc_mark_roots` walks.

This works but has a maintenance problem: every new opcode that allocates must correctly pin all live C locals, and forgetting to pin causes a use-after-free that only manifests under `GC_STRESS`. There are currently 46 pin/unpin call sites across ~15 opcodes.

A **shadow stack** is the standard alternative (used by Lua, Ruby, and others). Instead of pushing the object's current value, it pushes the **address of the C local variable** itself. When the GC runs, it dereferences each address to find the current object and marks it. This has one additional advantage: if a local is reassigned between the push and the GC, the GC sees the updated value automatically.

## Design

The shadow stack has two kinds of entries:

1. **`Obj**` entries** — address of a C local holding an `ObjArray*`, `ObjMap*`, `ObjClosure*`, etc. The GC dereferences to get the `Obj*` and marks it. This replaces `gc_pin((Obj *)ptr)`.

2. **`Value*` entries** — address of a C local holding a `Value`. The GC calls `gc_mark_value` on it. This replaces `gc_pin_value(&v)`.

The API:

```c
void gc_root_obj(void *local_addr);    // push &my_obj_ptr
void gc_root_value(Value *local_addr); // push &my_value
void gc_unroot(void);                  // pop one entry
void gc_unroot_n(int n);               // pop n entries (convenience)
```

The stack is a fixed-size array (matching `GC_TEMP_ROOTS_MAX`, currently 16). LIFO discipline, same as the current pin stack.

## Acceptance criteria

- [ ] Shadow stack data structures exist in `src/gc.c`: a tagged array of entries (each either `Obj**` or `Value*`).
- [ ] `gc_root_obj(void *local_addr)` pushes an `Obj**` entry onto the shadow stack.
- [ ] `gc_root_value(Value *local_addr)` pushes a `Value*` entry onto the shadow stack.
- [ ] `gc_unroot()` pops the most recent entry.
- [ ] `gc_unroot_n(int n)` pops `n` entries at once.
- [ ] `gc_mark_roots` walks the shadow stack: for `Obj**` entries, dereferences and calls `gc_mark_object`; for `Value*` entries, calls `gc_mark_value`.
- [ ] All `gc_pin` / `gc_pin_value` / `gc_unpin` calls in `src/vm.c` are replaced with the shadow stack equivalents.
- [ ] `gc_pin`, `gc_unpin`, `gc_pin_value` are removed from `src/gc.h` and `src/gc.c`.
- [ ] `GC_TEMP_ROOTS_MAX` is renamed or kept, used for the shadow stack capacity.
- [ ] `make test && make check` pass.
- [ ] `make test-sanitize` passes.
- [ ] `make test-gc-stress` passes.

## Dependencies

None. This is a self-contained refactor.

## Constraints and non-goals

- **Mechanical replacement.** Each `gc_pin((Obj *)x)` becomes `gc_root_obj(&x)`. Each `gc_pin_value(&v)` becomes `gc_root_value(&v)`. Each `gc_unpin()` becomes `gc_unroot()`. Sequences of N `gc_unpin()` calls can optionally become a single `gc_unroot_n(N)`.
- **No opcode logic changes.** The control flow and data flow within each opcode stays the same. Only the rooting calls change.
- **Fixed-size stack.** The shadow stack is a fixed-size array, same as today. If an opcode ever needs more than 16 roots, increase the constant.
- **Compiler roots unchanged.** `gc_push_compiler_root` / `gc_pop_compiler_root` are a separate mechanism for the compiler and are not touched by this task.

## Implementation steps

### Step 1: Write tests for the shadow stack API

Add tests to `tests/test_gc.c`:

- **`test_gc_root_obj_survives_collect`**: Allocate an ObjArray via `gc_alloc`. Pop it off any root set so it's unreachable. Call `gc_root_obj(&arr)` where `arr` is the local. Call `gc_collect()`. Verify the array survived (still on the object list and `is_marked == false` — meaning it was marked, survived sweep, and had its mark cleared).
- **`test_gc_root_value_survives_collect`**: Same pattern but with a `Value v = make_array(...)` and `gc_root_value(&v)`.
- **`test_gc_root_obj_reassigned_local`**: Push `gc_root_obj(&arr)`, then reassign `arr = gc_alloc(...)` (a new array). Call `gc_collect()`. Verify the *new* array survived (the shadow stack dereferenced &arr and found the updated pointer).
- **`test_gc_unroot_n`**: Push 3 roots, call `gc_unroot_n(3)`, verify the shadow stack is empty (allocate unreachable objects, collect, verify they were swept).

Run `make test` — new tests fail (shadow stack API doesn't exist yet).

### Step 2: Implement the shadow stack in gc.h/gc.c

In `src/gc.h`:
- Declare `gc_root_obj(void *local_addr)`, `gc_root_value(Value *local_addr)`, `gc_unroot(void)`, `gc_unroot_n(int n)`.
- Keep `GC_TEMP_ROOTS_MAX` (or rename to `GC_SHADOW_STACK_MAX`).

In `src/gc.c`:
- Define a tagged entry type:
  ```c
  typedef enum { GC_ROOT_OBJ_PTR, GC_ROOT_VALUE_PTR } GcRootKind;
  typedef struct {
      GcRootKind kind;
      void *addr; /* Obj** or Value* */
  } GcShadowEntry;
  ```
- Replace `temp_root_stack` / `temp_root_count` with `GcShadowEntry shadow_stack[GC_SHADOW_STACK_MAX]` / `int shadow_count`.
- Implement `gc_root_obj`: push `{GC_ROOT_OBJ_PTR, local_addr}`.
- Implement `gc_root_value`: push `{GC_ROOT_VALUE_PTR, local_addr}`.
- Implement `gc_unroot`: decrement `shadow_count`.
- Implement `gc_unroot_n`: decrement `shadow_count` by `n` (clamped to 0).
- Update `gc_mark_roots` — replace the old pin stack walk with:
  ```c
  for (int i = 0; i < shadow_count; i++) {
      if (shadow_stack[i].kind == GC_ROOT_OBJ_PTR) {
          Obj **ptr = (Obj **)shadow_stack[i].addr;
          if (*ptr != NULL)
              gc_mark_object(*ptr);
      } else {
          gc_mark_value((Value *)shadow_stack[i].addr);
      }
  }
  ```
- Reset the shadow stack in `gc_init`.

Keep the old `gc_pin`/`gc_unpin`/`gc_pin_value` functions for now (they will be removed in step 9 after all call sites are migrated). They can coexist during the transition — `gc_mark_roots` walks both stacks.

Run `make test && make check`.

### Step 3: Replace pinning in simple opcodes

Replace `gc_pin`/`gc_unpin`/`gc_pin_value` with shadow stack calls in these straightforward sites in `src/vm.c`:

- **`OP_ARRAY`**: `gc_pin((Obj *)arr)` → `gc_root_obj(&arr)`, `gc_unpin()` → `gc_unroot()`.
- **`OP_MAP`**: same pattern for `map`.
- **`native_keys`**: same pattern for `arr`.
- **`native_values`**: same pattern for `arr`.

Run `make test && make check && make test-gc-stress`.

### Step 4: Replace pinning in binary ops

### Step 5: Replace pinning in function calls

- **`OP_CALL` (native function path)**: `gc_pin_value(&callee_val)` → `gc_root_value(&callee_val)`, loop pinning `args[i]` → loop rooting.
