# GC Task 2: ObjString — GC-Tracked String Type

## Objective

Introduce `ObjString` as a GC-tracked heap object that wraps string data. Strings become first-class GC objects on the object list, replacing the current bare `char*` representation for `VAL_STRING` values. The `Value` struct is updated so `VAL_STRING` holds an `ObjString*` instead of `char*`. `VAL_ERROR` continues to use `char*` (errors are transient and don't need GC tracking).

No string interning yet — each string creation produces a new `ObjString`. Interning requires the sweep phase (task 4) to clean the intern table. For now, `value_clone` of a string creates a new `ObjString` with a `strdup`'d copy, and `value_free` frees it (same ownership semantics as today, just wrapped in ObjString).

When done:

- `ObjString` struct exists with `Obj` header, `char *chars`, `size_t length`, `uint32_t hash`.
- `OBJ_STRING` is added to the `ObjType` enum.
- `obj_string_new(const char *chars, size_t length)` allocates through `gc_alloc`, copies the string, and computes the hash.
- `obj_string_take(char *chars, size_t length)` takes ownership of an existing `char*` buffer.
- The `Value` struct has a new `ObjString *string` field for `VAL_STRING`; `VAL_ERROR` uses a separate `char *error` field.
- All string creation, cloning, freeing, comparison, and access sites are updated.
- `make test && make check` pass.

- [ ] `value_clone()` for `VAL_ERROR`: `strdup` on `error` field (unchanged semantics).
- [ ] `value_free()` for `VAL_STRING`: calls `gc_unlink` + frees `chars` + frees ObjString (same refcount-free pattern as current).
- [ ] `value_free()` for `VAL_ERROR`: `free(v->error)`.

- [ ] All sites in `src/vm.c`, `src/compiler.c`, `src/chunk.c`, `src/json.c`, `src/repl.c` that access `v->string` for `VAL_STRING` are updated to `v->string->chars`.
- [ ] `gc_free_all()` in `src/gc.c` handles `OBJ_STRING`: frees `chars`, then the ObjString.
- [ ] Tests in `tests/test_value.c` updated for the new string representation.
- [ ] `make test && make check` pass.

## Dependencies

- **gc-infrastructure** (task 1): `Obj` header, `gc_alloc`, `gc_unlink`, `ObjType` enum.

## Constraints and non-goals

- **No string interning.** Each `value_clone` of a string creates a new ObjString. Interning is added in task 4 when sweep can clean the intern table.
- **No change to string comparison semantics.** Still uses `strcmp` (content equality). Pointer equality optimization comes with interning.
- **Refcounting stays.** ObjString is allocated on the GC list but freed via refcount-like ownership in `value_free`. This is a transitional state.
- **VAL_ERROR stays simple.** Errors use `char *error` (not ObjString). Errors are transient, always uniquely owned, and don't benefit from GC tracking.
- **The `make_string` API may change.** Currently `make_string(char *s)` takes ownership of a raw `char*`. The new version may take an `ObjString*` or wrap the call to `obj_string_take` internally. Choose whatever minimizes churn at call sites.

### Step 1: Write tests for ObjString

Add tests to `tests/test_value.c` (or a new `tests/test_gc.c` section):

- **`test_obj_string_new`**: Create an ObjString via `obj_string_new("hello", 5)`, verify `chars` equals `"hello"`, `length` is 5, `hash` is non-zero, and `OBJ_TYPE(str) == OBJ_STRING`.
- **`test_obj_string_take`**: `strdup("world")`, pass to `obj_string_take`, verify it wraps correctly.
- **`test_obj_string_hash_consistency`**: Two ObjStrings with the same content have the same hash.
- **`test_obj_string_hash_different`**: Two ObjStrings with different content have different hashes (probabilistic, use known distinct strings).
- **`test_make_string_returns_val_string`**: `make_string(...)` returns a Value with `type == VAL_STRING` and `v.string->chars` matching the input.
- **`test_value_clone_string_independent`**: Clone a string Value, modify the original's content (by freeing and replacing), verify the clone is independent.
- **`test_value_free_string`**: Create and free a string Value, verify no leaks (rely on sanitizers).
- **`test_value_equal_strings`**: Two string Values with same content are equal.
- **`test_make_error_uses_error_field`**: `make_error("oops")` produces a Value with `type == VAL_ERROR` and `v.error` containing `"oops"`.

### Step 3: Update the Value struct — split string and error fields

In `src/value.h`, update the `Value` struct:

```
struct Value {
    ValueType type;
    double number;
    bool boolean;
    ObjString *string;     /* owned; non-NULL only for VAL_STRING */
    char *error;           /* owned; non-NULL only for VAL_ERROR */
    ObjFunction *function;
    ObjClosure *closure;
    ObjArray *array;
    ObjMap *map;
};
```

This splits the old `char *string` (shared by VAL_STRING and VAL_ERROR) into:
- `ObjString *string` for VAL_STRING
- `char *error` for VAL_ERROR

This will break compilation at every site that accesses `v->string` for errors. Fix all error-related access sites:

**In `src/value.c`**:
- `make_error()`: Change `.string = strdup(buf)` to `.error = strdup(buf)`.
- `value_free()`: The `if (v->string)` block that frees the string for both types needs splitting. For VAL_ERROR: `free(v->error); v->error = NULL;`. For VAL_STRING: will be handled separately (see step 4).
- `value_format()` for `VAL_ERROR`: Change `v->string` to `v->error`.
- `value_clone()` for `VAL_ERROR`: Change to `out->error = strdup(src->error ? src->error : "")`.

**In `src/vm.c`** and other files: Search for all access to `v->string` on VAL_ERROR paths. The error message is often accessed via `result.string` in error-handling code. Update all to `result.error`.

**In `src/repl.c`**, `src/json.c`**, `src/main.c`**: Any code that reads the error message from a Value (e.g., formatting error responses) must use `v->error` instead of `v->string`.

Run `make` to verify compilation.

### Step 4: Update make_string and VAL_STRING handling

**Update `make_string()`** in `src/value.c`:

The current signature is `Value make_string(char *s)` which takes ownership of a `char*`. Change it to wrap the string in an ObjString:

```c
Value make_string(char *s) {
    size_t len = s ? strlen(s) : 0;
    ObjString *str = obj_string_take(s, len);
    return (Value){.type = VAL_STRING, .string = str};
}
```

This preserves the existing API: callers pass a heap-allocated `char*`, and `make_string` takes ownership (via `obj_string_take`).

Add a new convenience: `make_string_copy(const char *s, size_t len)` that copies instead of taking ownership (uses `obj_string_new` internally). This is useful for sites that have a `const char*`.

**Update `value_free()`** for VAL_STRING:
- Free `v->string->chars`, call `gc_unlink((Obj*)v->string)`, `free(v->string)`, set `v->string = NULL`.

**Update `value_clone()`** for VAL_STRING:
- Create a new ObjString via `obj_string_new(src->string->chars, src->string->length)`.
- Set `out->string = new_str`.

**Update `value_equal()`** for VAL_STRING:
- Change `strcmp(a->string, b->string)` to `strcmp(a->string->chars, b->string->chars)`.
- Optimization: if `a->string == b->string`, return true immediately (same object).
- Optimization: if `a->string->hash != b->string->hash`, return false (different hashes means different content).

**Update `value_format()`** for VAL_STRING:
- Change `strdup(v->string ? v->string : "")` to `strdup(v->string ? v->string->chars : "")`.

**Update `is_truthy()`** for VAL_STRING:
- Change `v->string != NULL && v->string[0] != '\0'` to `v->string != NULL && v->string->length > 0`.

Run `make` to verify compilation.

### Step 5: Update all VAL_STRING access sites across the codebase

Search for every remaining access to `v->string` (or `->string`) in files other than `value.c` and `value.h`. These are sites that read the string content and need to go through `->chars`:

**`src/vm.c`**: The VM accesses string values in many opcodes:
- `OP_CONCAT`: string concatenation reads `a.string` and `b.string` to build a new string. Update to `a.string->chars` and `b.string->chars` (also use `a.string->length`).
- `OP_ADD` (string+number coercion if any).
- `OP_IN` (string-in-string search via `strstr`).
- `OP_INDEX_GET` / `OP_INDEX_SET` for string indexing.
- `native_say()`, `native_str()`, `native_len()` and other native functions.
- `value_type_name()` (doesn't access string content, OK).
- Error message formatting with string values.

**`src/compiler.c`**:
- `compiler_strdup()` and constant pool string handling. The compiler creates `VAL_STRING` values for the constant pool via `make_string(compiler_strdup(...))`. This should work with the updated `make_string`.

**`src/chunk.c`**:
- `chunk_find_or_add_constant()`: Compares string constants via `strcmp`. Update to compare `->string->chars`.
- `chunk_disassemble` / `disassemble_instruction_to_buf`: May print string constants.

**`src/json.c`**: String serialization for REPL protocol.

**`src/repl.c`**: Accesses result values that may be strings.

For each file, search for patterns like `\.string` and `->string` where the Value type is `VAL_STRING`, and add `->chars` as needed.
