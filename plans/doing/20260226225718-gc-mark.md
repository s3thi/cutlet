# GC Task 3: Mark Phase â€” Root Walking and Reachability Tracing

## Objective

Implement the mark phase of mark-and-sweep garbage collection. `gc_collect()` walks all root sets (VM stack, open upvalues, global variable table, call frame closures, constant pools reachable from executing functions), marks every reachable object, and recursively traces references from marked objects to discover transitively reachable objects.

After this task, `gc_collect()` performs marking (but not sweeping â€” sweep is task 4). A `GC_STRESS` compile-time flag triggers `gc_collect()` on every `gc_alloc()` call to help catch missing roots during development.

When done:

- `gc_mark_value(Value *v)` marks the object inside a Value (if any) and recursively traces its children.
- `gc_mark_object(Obj *obj)` sets `is_marked = true` and recursively traces the object's references.
- `gc_mark_roots()` walks all root sets and marks everything reachable.
- `gc_collect()` calls `gc_mark_roots()` (sweep is a no-op until task 4).
- `GC_STRESS` mode calls `gc_collect()` on every allocation.
- A `gc_set_vm(VM *vm)` function stores a pointer to the currently executing VM so `gc_mark_roots` can walk its stack.
- `make test && make check` pass.

## Acceptance criteria

- [ ] `gc_mark_value(Value *v)` in `src/gc.c` marks the heap object inside a Value.
- [ ] `gc_mark_object(Obj *obj)` in `src/gc.c` sets `is_marked = true` and traces children:
  - `OBJ_FUNCTION`: marks nothing extra (name/params are not GC objects; chunk constants are traced separately).
  - `OBJ_CLOSURE`: marks `function`, marks each upvalue in the upvalue array.
  - `OBJ_ARRAY`: marks each element Value in `data`.
  - `OBJ_MAP`: marks each key and value in `entries`.
  - `OBJ_UPVALUE`: if closed, marks the `closed` Value.
  - `OBJ_STRING`: marks nothing extra (leaf object).
- [ ] `gc_mark_roots()` in `src/gc.c` walks:
  - VM value stack: `vm->stack[0]` through `vm->stack_top - 1`.
  - Open upvalue list: `vm->open_upvalues` linked list.
  - Global variable table: every Value in `var_table` (via a new `runtime_mark_globals()` helper).
  - Call frame closures: `vm->frames[i].closure` for each active frame (including frame 0 / the script closure).
  - Chunk constants: transitively reachable through marked ObjFunctions.
- [ ] `gc_set_vm(VM *vm)` stores the current VM pointer in the GC state. Called at `vm_execute()` entry and cleared on exit.
- [ ] `runtime_mark_globals()` in `src/runtime.c` iterates `var_table` and calls `gc_mark_value` on each stored Value.
- [ ] `gc_collect()` calls `gc_mark_roots()` and resets `is_marked` on all objects after marking (temporary, until sweep in task 4).
- [ ] `GC_STRESS` compile-time flag: when defined, `gc_alloc()` calls `gc_collect()` before every allocation.
- [ ] Tests in `tests/test_gc.c` verify marking behavior.
- [ ] `make test && make check` pass.

## Dependencies

- **gc-infrastructure** (task 1): `Obj` header, `gc_alloc`, `gc_unlink`, object list.
- **gc-objstring** (task 2): `OBJ_STRING` type, `ObjString` in the object list.

## Constraints and non-goals

- **No sweep phase.** After marking, all marks are cleared (reset). No objects are freed by the GC in this task. Sweep is task 4.
- **No GC triggering in production.** `gc_collect()` only runs when explicitly called (tests) or under `GC_STRESS`. Automatic threshold-based triggering is task 4.
- **Compiler temporaries are NOT roots.** The compiler runs to completion before `vm_execute()`, so no GC can trigger during compilation. If `eval()` is added later, this assumption must be revisited.
- **Stack-allocated script objects.** The `script_fn` and `script_closure` in `vm_execute()` are on the C stack, not the GC object list. They are accessible via `vm->frames[0].closure`, which `gc_mark_roots` traces. The frame closure pointer traces to the stack-allocated `script_closure`, which contains a pointer to `script_fn`. Since these are not on the GC list, `gc_mark_object` will encounter them. Handle this by either: (a) not calling `gc_mark_object` on objects not on the GC list (check a flag), or (b) putting them on the GC list too. Option (a) is simpler: add an `on_gc_list` flag to `Obj`, set by `gc_alloc`, checked by `gc_mark_object`. Or, more simply, check if `obj->next == NULL && obj != gc.objects` to detect non-list objects. The cleanest approach: `gc_mark_object` should be safe to call on any `Obj*` â€” it just sets `is_marked = true` and traces children. The sweep phase (task 4) only walks the GC list, so non-list objects are never swept.

### Step 2: Implement gc_mark_object and gc_mark_value

In `src/gc.c`:

**`gc_mark_object(Obj *obj)`**:
- If `obj == NULL` or `obj->is_marked`, return (prevents infinite loops on cycles).
- Set `obj->is_marked = true`.
- Switch on `obj->type`:
  - `OBJ_STRING`: No children to trace. Return.
  - `OBJ_UPVALUE`: Cast to `ObjUpvalue*`. If closed (`location == &closed`), call `gc_mark_value(&uv->closed)`.
  - `OBJ_FUNCTION`: Cast to `ObjFunction*`. The function owns a `Chunk*` whose constant pool may contain ObjFunctions and ObjStrings. Iterate `fn->chunk->constants[0..const_count-1]` and call `gc_mark_value` on each. (If `fn->chunk == NULL`, it's a native function â€” nothing to trace.)
  - `OBJ_CLOSURE`: Cast to `ObjClosure*`. Call `gc_mark_object((Obj*)cl->function)`. For each upvalue `cl->upvalues[i]`, if non-NULL, call `gc_mark_object((Obj*)cl->upvalues[i])`.
  - `OBJ_ARRAY`: Cast to `ObjArray*`. For each element `arr->data[i]` (i < count), call `gc_mark_value(&arr->data[i])`.
  - `OBJ_MAP`: Cast to `ObjMap*`. For each entry (i < count), call `gc_mark_value(&m->entries[i].key)` and `gc_mark_value(&m->entries[i].value)`.

**`gc_mark_value(Value *v)`**:
- Switch on `v->type`:
  - `VAL_FUNCTION`: `gc_mark_object((Obj*)v->function)`.
  - `VAL_CLOSURE`: `gc_mark_object((Obj*)v->closure)`.
  - `VAL_ARRAY`: `gc_mark_object((Obj*)v->array)`.
  - `VAL_MAP`: `gc_mark_object((Obj*)v->map)`.
  - `VAL_STRING`: `gc_mark_object((Obj*)v->string)`.
  - All others (number, bool, nothing, error): no-op.

Declare both functions in `src/gc.h`.

Run `make test && make check`.

### Step 3: Implement gc_set_vm and gc_mark_roots

**`gc_set_vm(VM *vm)`** in `src/gc.c`:
- Store `vm` in a `static VM *gc_vm` variable.
- Declare in `src/gc.h`.

**In `src/vm.c`**:
- At the start of `vm_execute()`, after creating the VM and setting up frame 0, call `gc_set_vm(&vm)`.
- At the end of `vm_execute()` (all return paths â€” success and error), call `gc_set_vm(NULL)`.
- Add `#include "gc.h"`.

**`gc_mark_roots()`** in `src/gc.c`:
- If `gc_vm != NULL`:
  - **VM stack**: iterate from `gc_vm->stack` to `gc_vm->stack_top - 1`, call `gc_mark_value` on each.
  - **Call frames**: iterate `gc_vm->frames[0..frame_count-1]`, for each frame call `gc_mark_object((Obj*)frame->closure)`.
  - **Open upvalues**: walk `gc_vm->open_upvalues` linked list, call `gc_mark_object((Obj*)uv)` for each.
- **Globals**: call `runtime_mark_globals()` (always, even if no VM is active â€” globals persist across evaluations).

Run `make test && make check`.

### Step 5: Update gc_collect and add GC_STRESS mode

**Update `gc_collect()`** in `src/gc.c`:
- Call `gc_mark_roots()`.
- After marking, walk the entire object list (`gc.objects`) and reset `is_marked = false` on every object. (This is temporary â€” once sweep is added in task 4, the reset happens as part of sweep for surviving objects.)
- Add a `/* TODO: sweep phase â€” see gc-sweep task */` comment.

**Add `GC_STRESS` mode**:
- In `gc_alloc()`, add: `#ifdef GC_STRESS` / `gc_collect();` / `#endif` at the start of the function (before the actual allocation).
- In the Makefile, add a `test-gc-stress` target that compiles and runs the test suite with `-DGC_STRESS`. This is a development-time check, not part of the regular `make test`.

Run `make test && make check`. Also run `make test-gc-stress` (if stress mode uncovers bugs, fix them before proceeding).

### Step 6: Handle edge cases and run full verification

**Edge cases to verify**:

- **Empty VM stack**: `gc_mark_roots` when `gc_vm == NULL` (no active evaluation) should only mark globals.
- **Native functions**: Native ObjFunctions have `chunk == NULL`. `gc_mark_object` for `OBJ_FUNCTION` must check `fn->chunk != NULL` before iterating constants.
- **Deeply nested closures**: A closure capturing upvalues that reference other closures forms a reference chain. Marking should trace the entire chain without stack overflow. For typical depths this is fine; add a comment noting that very deep reference chains could overflow the C stack and that an explicit worklist could be added later if needed.
- **Stack-allocated script objects**: `vm->frames[0].closure` points to the stack-allocated `script_closure`. `gc_mark_object` will try to set its `is_marked` â€” this is harmless since it's never swept (it's not on the GC list). Verify this doesn't cause issues.

- `make check` â€” linting passes.
- `make test-sanitize` â€” no memory issues.
- `make test-gc-stress` â€” no crashes under stress.
