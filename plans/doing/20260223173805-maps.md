# Maps (Dictionaries) — Core Data Type

## Objective

Add a map/dictionary data type to Cutlet with literal syntax, key-based indexing, map projection via `[]` with an array of keys, merge via `++`, builtin functions (`keys()`, `values()`, `has_key()`, `len()` extended), structural equality, and value semantics with COW refcounting.

When done:

- Map literals work: `{}`, `{name: "alice", age: 30}`, `{[1 + 2]: "three"}`.
- Bracket indexing works: `m["key"]`, `m["key"] = value`. Missing keys return `nothing`.
- `map[array_of_keys]` returns a projected sub-map.
- `++` merges two maps (right-hand side wins on key conflicts).
- `keys(m)`, `values(m)`, `has_key(m, k)`, `len(m)` work.
- Maps use value semantics with COW refcounting (same as arrays).
- Structural equality: `{a: 1, b: 2} == {b: 2, a: 1}` is true.
- `make test && make check` pass.

## Acceptance criteria

- [ ] `{`, `}`, `:` tokenize as solo-symbol operators.
- [ ] `ObjMap` struct with refcounting and COW in `src/value.h` / `src/value.c`.
- [ ] `VAL_MAP` in the `ValueType` enum; `make_map()`, `value_format()`, `value_free()`, `value_clone()`, `is_truthy()` handle maps.
- [ ] `bool value_equal(const Value *a, const Value *b)` extracted and usable by both `ObjMap` and the VM.
- [ ] `AST_MAP` node; parser handles bare-identifier keys, expression keys, computed `[expr]` keys, trailing commas, empty maps.
- [ ] `OP_MAP` opcode builds a map from stack values.

- [ ] `OP_INDEX_GET` handles map + array-of-keys → projected sub-map.
- [ ] `OP_CONCAT` handles map-map merge (right side wins).
- [ ] `keys()`, `values()`, `has_key()` registered as native functions; `len()` extended for maps.
- [ ] Structural equality for maps in `values_equal()`.
- [ ] Parser tests, value tests, VM integration tests all pass.
- [ ] `make test && make check` pass.

## Dependencies

- **Arrays plan** (`plans/doing/20260219150840-arrays.md`): must be complete. This plan requires `VAL_ARRAY`, `ObjArray`, `OP_ARRAY`, `OP_INDEX_GET`, `OP_INDEX_SET`, `OP_CONCAT` (array support), `len()` native, and the array value infrastructure to be in place.

## Constraints and non-goals

- **No dot access.** `map.key` sugar is deferred to a follow-up plan.
- **No `for` loop.** Iteration is deferred. Use `keys()`/`values()` + while loops.
- **No JSON parsing.** `json_parse()`/`json_format()` are separate follow-up plans.
- **No `@` meta-operators on maps.** Covered by a separate plan (`map-meta-operators`).
- **No `@:` zip.** Covered by a separate plan (`map-zip`).
- **No ordered iteration guarantee in the language spec** — but the implementation preserves insertion order for deterministic `keys()`/`values()` output and testability.
- **No deletion.** There is no `delete(map, key)` or similar in this plan. Defer to a follow-up.
- **Key types:** strings, numbers, booleans, and `nothing` are valid keys. Functions and errors are not valid keys (runtime error on attempt).

## Design

### Map literals

```cutlet
{}                                # empty map
{name: "alice", age: 30}         # bare identifiers are string keys
{"key": "value"}                  # string expression keys
{[1 + 2]: "three"}               # computed key (any expression)
{[true]: "yes", [false]: "no"}   # non-string keys via computed syntax
{a: 1, b: 2,}                    # trailing comma allowed
```

Bare identifier keys: when the parser sees `IDENT :` inside `{...}`, the identifier is treated as a string literal key (not a variable reference). To use a variable as a key, use the computed form: `{[my_var]: value}`.

### Map indexing

```cutlet
my m = {name: "alice", age: 30}
m["name"]            # => alice
m["age"]             # => 30
m["missing"]         # => nothing  (not an error)
m["name"] = "bob"    # update existing key
m["email"] = "a@b"   # insert new key (COW if shared)
```

Missing-key-returns-`nothing` is deliberate: it enables `if m["key"] then ...` guard patterns without try/catch. Use `has_key(m, k)` to distinguish "key absent" from "key present with value `nothing`".

### Map projection with `[]`

When the index into a map is an array, it selects a sub-map:

```cutlet
my person = {name: "alice", age: 30, email: "a@b.com", role: "admin"}
person[["name", "email"]]
# => {name: "alice", email: "a@b.com"}
```

Keys in the array that don't exist in the map are silently skipped (no error). This parallels array boolean-mask indexing: arrays filter by position, maps filter by key.

### Value semantics (COW)

Same as arrays. `ObjMap` is refcounted. Assignment increments the refcount (cheap shallow copy). Mutation (index-set, merge) triggers copy-on-write if refcount > 1.

```cutlet
my m1 = {a: 1}
my m2 = m1           # m2 shares backing store (refcount 2)
m2["b"] = 2          # COW: m2 gets its own copy
m1                    # => {a: 1} (unchanged)
```

### Map merge with `++`

```cutlet
{a: 1, b: 2} ++ {b: 3, c: 4}   # => {a: 1, b: 3, c: 4}
{} ++ {a: 1}                     # => {a: 1}
{a: 1} ++ {}                     # => {a: 1}
```

Right-hand side wins on key conflicts. Both operands must be maps (map ++ non-map is an error). When neither operand is a map or array, existing string/array concatenation behavior is unchanged.

### Builtin functions

```cutlet
len({a: 1, b: 2})              # => 2
len({})                         # => 0
keys({name: "alice", age: 30}) # => ["name", "age"]  (insertion order)
values({name: "alice", age: 30}) # => ["alice", 30]  (insertion order)
has_key({a: 1}, "a")           # => true
has_key({a: 1}, "b")           # => false
has_key({a: nothing}, "a")     # => true  (key exists, value is nothing)
```

`keys()` and `values()` return arrays (require `VAL_ARRAY` / `ObjArray` from the arrays plan). `has_key()` takes a map and a key, returns a boolean.

### Equality

Structural equality: two maps are equal iff they have the same set of keys and all values are pairwise equal. Key order does not matter for equality.

```cutlet
{a: 1, b: 2} == {b: 2, a: 1}   # => true
{a: 1} == {a: 1, b: 2}          # => false
{a: 1} == "hello"                # => false (different types)
{} == {}                          # => true
```

### Formatting

`value_format()` for maps: `{key1: value1, key2: value2}`. Keys and values formatted with `value_format()`. Empty map: `{}`. Insertion order preserved.

```
{name: alice, age: 30}    # string values unquoted (matches array element formatting)
{1: one, true: yes}       # non-string keys use value_format()
```

### Truthiness

Non-empty maps are truthy, empty maps `{}` are falsy. (Matches arrays: non-empty = truthy, empty = falsy.)

## Architecture changes

### `ObjMap` backing store

```c
/* A single key-value entry in a map. */
typedef struct {
    Value key;    /* Owned key. */
    Value value;  /* Owned value. */
} MapEntry;

/* Heap-allocated backing store for maps. Reference-counted.
 * Entries are stored in a dense array in insertion order.
 * Lookup is O(n) linear scan — sufficient for typical shell-script
 * map sizes. Can be upgraded to a hash index later without API changes. */
typedef struct {
    size_t refcount;    /* Reference count (1 on creation). */
    MapEntry *entries;  /* Owned array of key-value pairs. */
    size_t count;       /* Number of live entries. */
    size_t capacity;    /* Allocated capacity of entries array. */
} ObjMap;
```

Key equality for lookup uses a standalone `value_equal()` function in `value.c`, extracted from the VM's `values_equal()`. This avoids a circular dependency between `value.c` and `vm.c`.

### New token handling

`{`, `}`, and `:` become solo symbols in `is_solo_symbol()`. No new `TokenType` needed — they emit as `TOK_OPERATOR` with values `"{"`, `"}"`, `":"`.

### New AST node: `AST_MAP`

| Node | Fields used | Purpose |
|------|------------|---------|
| `AST_MAP` | `children`, `child_count` | Map literal `{key: value, ...}` |

`children` contains alternating key-value expression nodes: `[key0, val0, key1, val1, ...]`. `child_count` is always even (2 * number of entries). For bare identifier keys, the parser creates an `AST_STRING` node with the identifier's text as the value.

S-expression format:
- `[MAP]` for empty map.
- `[MAP [STRING name] [NUMBER 42] [STRING age] [NUMBER 30]]` for `{name: 42, age: 30}`.

### New opcode

| Opcode | Operand | Stack effect | Description |
|--------|---------|-------------|-------------|
| `OP_MAP` | 1-byte count (number of pairs) | pop 2N, push 1 | Build map from top 2N stack values (key, value pairs) |

`OP_INDEX_GET`, `OP_INDEX_SET`, and `OP_CONCAT` are extended (not new) — the VM handlers check the value type and dispatch accordingly.

### Step 2: `VAL_MAP` value type with `ObjMap`

Add the map value type with `ObjMap` backing store and reference counting.

**Value changes** (`src/value.h`, `src/value.c`):
- Add `ObjMap` and `MapEntry` struct definitions (as shown in architecture section).
- Add `VAL_MAP` to the `ValueType` enum (after `VAL_CLOSURE`).
- Add `ObjMap *map` field to the `Value` struct.
- Constructors:
  - `ObjMap *obj_map_new(void)` — allocate empty map, refcount 1.
  - `void obj_map_set(ObjMap *m, const Value *key, const Value *val)` — insert or update. Clones key and value. If key already exists, update in place.
  - `Value *obj_map_get(const ObjMap *m, const Value *key)` — return pointer to value for key, or NULL if not found. Does not clone.
  - `bool obj_map_has(const ObjMap *m, const Value *key)` — return true if key exists.
  - `ObjMap *obj_map_clone_deep(const ObjMap *src)` — full independent copy with refcount 1.
  - `void obj_map_ensure_owned(Value *v)` — COW: if refcount > 1, deep-clone and replace.
  - `Value make_map(ObjMap *m)` — wrap ObjMap in Value.
- Add `bool value_equal(const Value *a, const Value *b)` — extracted from `values_equal()` in `vm.c`. Used by `obj_map_get()` and `obj_map_set()` for key comparison, and by the VM for map equality. Handles `VAL_NUMBER`, `VAL_STRING`, `VAL_BOOL`, `VAL_NOTHING`, `VAL_FUNCTION`, `VAL_CLOSURE`. Does NOT handle `VAL_ARRAY` or `VAL_MAP` yet (those are added by the array and map equality steps respectively).
- `value_format()`: `"{key1: value1, key2: value2}"` format, `"{}"` for empty. Keys and values formatted with `value_format()`.
- `value_free()`: decrement `m->refcount`. If it reaches 0, `value_free()` each key and value in entries, then free `entries`, then free the ObjMap.
- `value_clone()`: `out->map = src->map; src->map->refcount++`.
- `is_truthy()`: `m->count > 0`.

**Tests** (`tests/test_value.c` or wherever value tests live):
- Create an empty ObjMap, wrap in Value, format → `"{}"`.
- Create a map with string keys `{a: 1, b: 2}`, format → `"{a: 1, b: 2}"`.
- `obj_map_get` for existing key returns pointer to value.
- `obj_map_get` for missing key returns NULL.
- `obj_map_set` with duplicate key updates the value.
- `obj_map_has` returns true/false correctly.
- Clone a VAL_MAP → refcount is 2. Free clone → refcount is 1. Free original → ObjMap freed.
- `is_truthy({a: 1})` → true. `is_truthy({})` → false.
- `obj_map_ensure_owned` with refcount 1 → no-op.
- `obj_map_ensure_owned` with refcount 2 → deep clone, original refcount drops to 1.
- `value_equal` works for numbers, strings, bools, nothing (unit test the extracted function).
- Mixed-type keys: `{[1]: "one", [true]: "yes"}` format → `"{1: one, true: yes}"`.

**Files touched**: `src/value.h`, `src/value.c`, tests.

### Step 3: Map literals (parser + compiler + VM)

Parse `{key: value, ...}`, compile to `OP_MAP`, execute in VM.

**AST** (`src/parser.h`):
- Add `AST_MAP` to `AstNodeType`.

**Parser** (`src/parser.c`):
- Register `{` as a prefix parse rule (like `[` for arrays).
- `parse_map_literal()`: consume `{`. If next token is `}`, return empty map. Otherwise:
  - **Bare key**: if the current token is `TOK_IDENT` and the token after it is `:`, treat the identifier as a string key — create an `AST_STRING` child node with the identifier text. Consume the ident and `:`.
  - **Computed key**: if the current token is `[`, consume `[`, parse an expression for the key, consume `]`, then consume `:`.
  - **Expression key**: otherwise parse an expression for the key, then consume `:`.
  - Parse an expression for the value.
  - If next token is `,`, consume it and continue (allow trailing comma before `}`).
  - Store key and value expressions as alternating children in `children` array.
  - Consume `}`.
- `ast_node_type_str()`: `AST_MAP` → `"MAP"`.
- `ast_format_node()`: `"[MAP [key1] [val1] [key2] [val2] ...]"`, `"[MAP]"` for empty.
- `ast_free()`: free each child.
- `parser_is_complete()`: track `{` as a bracket that requires `}` (like `(` and `[`).

**Opcode** (`src/chunk.h`):
- Add `OP_MAP` with 1-byte count operand (number of key-value pairs, not number of stack values).

**Compiler** (`src/compiler.c`):
- `compile_map()`: for each pair, compile key expression (push key), compile value expression (push value). After all pairs, emit `OP_MAP [pair_count]`.
- Add `case AST_MAP: compile_map(c, node); break;` to the switch in `compile_node`.

**VM** (`src/vm.c`):
- `OP_MAP`: read count byte (number of pairs). Pop `2 * count` values from stack (value, key, value, key — reverse order since first pair was pushed first). Validate each key is a valid key type (string, number, bool, nothing — not function/closure/error). Create new ObjMap, insert pairs in order. Push resulting `VAL_MAP`.
- Add `"map"` to `value_type_name()` for `VAL_MAP`.

**Disassembler** (`src/chunk.c`):
- Format `OP_MAP` with pair count.

**Tests**:
- Parse `{}` → `[MAP]`.
- Parse `{a: 1, b: 2}` → `[MAP [STRING a] [NUMBER 1] [STRING b] [NUMBER 2]]`.
- Parse `{[1 + 2]: "three"}` → `[MAP [BINOP + [NUMBER 1] [NUMBER 2]] [STRING three]]`.
- Parse `{a: 1,}` → same as `{a: 1}` (trailing comma).
- Eval `{}` → `{}`.
- Eval `{a: 1, b: 2}` → `{a: 1, b: 2}`.
- Eval `{[1 + 2]: "three"}` → `{3: three}`.
- Eval `{[true]: "yes", [false]: "no"}` → `{true: yes, false: no}`.
- Eval `{a: 1, a: 2}` → `{a: 2}` (last value wins for duplicate keys).
- `parser_is_complete("{a: 1")` → false.
- `parser_is_complete("{a: 1}")` → true.
- Error: key is a function → runtime error "invalid map key type: function".

**Files touched**: `src/parser.h`, `src/parser.c`, `src/chunk.h`, `src/chunk.c`, `src/compiler.c`, `src/vm.c`, tests.

### Step 4: Map indexing (read + write) + projection

Extend `OP_INDEX_GET` and `OP_INDEX_SET` to handle maps.

**VM** (`src/vm.c`):
- In `OP_INDEX_GET`: after the existing array-index path (checking for `VAL_ARRAY`), add a `VAL_MAP` path:
  - If the indexed value is `VAL_MAP` and the index is a `VAL_ARRAY`: **projection**. Create a new ObjMap. For each key in the index array, if the key exists in the source map, add it to the result. Push the result map.
  - If the indexed value is `VAL_MAP` and the index is not an array: **key lookup**. Validate the key is a valid key type. Call `obj_map_get()`. If found, clone and push the value. If not found, push `nothing`.
- In `OP_INDEX_SET`: after the existing array path, add a `VAL_MAP` path:
  - Validate the key is a valid key type. Call `obj_map_ensure_owned()` (COW). Call `obj_map_set()` with the key and value. Push the value as expression result.

**Tests**:
- `{a: 1, b: 2}["a"]` → 1.
- `{a: 1, b: 2}["c"]` → nothing (missing key).
- `{[1]: "one"}[1]` → `"one"` (number key).
- `{[true]: "yes"}[true]` → `"yes"` (boolean key).
- `my m = {a: 1}\nm["a"] = 99\nm` → `{a: 99}`.
- `my m = {a: 1}\nm["b"] = 2\nm` → `{a: 1, b: 2}` (insert new key).
- COW: `my m1 = {a: 1}\nmy m2 = m1\nm2["a"] = 99\nm1` → `{a: 1}` (m1 unchanged).
- Projection: `{a: 1, b: 2, c: 3}[["a", "c"]]` → `{a: 1, c: 3}`.
- Projection with missing keys: `{a: 1, b: 2}[["a", "z"]]` → `{a: 1}` (z silently skipped).
- Projection with empty array: `{a: 1}[[]]` → `{}`.
- Error: invalid key type: `{a: 1}[fn(x) is x end]` → runtime error.

**Files touched**: `src/vm.c`, tests.

### Step 5: Map merge with `++`

Extend `OP_CONCAT` to handle map-map merging.

**VM** (`src/vm.c`):
- In the `OP_CONCAT` handler: before the existing string logic, add handling for maps. Check the type of both operands:
  - If **both** are `VAL_MAP`: create a new ObjMap. Copy all entries from the left map, then insert/overwrite with all entries from the right map. Push the result.
  - If one is a map and the other is not → runtime error "cannot concatenate map with \<type\>".
  - Existing array and string paths remain unchanged.

**Tests**:
- `{a: 1, b: 2} ++ {b: 3, c: 4}` → `{a: 1, b: 3, c: 4}`.
- `{} ++ {a: 1}` → `{a: 1}`.
- `{a: 1} ++ {}` → `{a: 1}`.
- `{} ++ {}` → `{}`.
- `{a: 1} ++ 2` → error "cannot concatenate map with number".
- `"a" ++ {b: 1}` → error "cannot concatenate string with map".
- `[1] ++ {a: 1}` → error "cannot concatenate array with map".
- `"a" ++ "b"` → `"ab"` (unchanged).

**Files touched**: `src/vm.c`, tests.

### Step 6: Builtin functions — `keys()`, `values()`, `has_key()`, extend `len()`

Register `keys`, `values`, `has_key` as native functions. Extend the existing `len` (added by the arrays plan) to handle maps.

**Native functions** (in `vm.c` near `register_builtins`):
- `native_keys(argc, args, ctx)`: validate argc == 1, args[0] is `VAL_MAP`. Create a new `ObjArray`, push each key (cloned) from the map in insertion order. Return `make_array(arr)`.
- `native_values(argc, args, ctx)`: validate argc == 1, args[0] is `VAL_MAP`. Create a new `ObjArray`, push each value (cloned) from the map in insertion order. Return `make_array(arr)`.
- `native_has_key(argc, args, ctx)`: validate argc == 2, args[0] is `VAL_MAP`, args[1] is a valid key type. Return `make_bool(obj_map_has(args[0].map, &args[1]))`.
- Extend `native_len()` (from the arrays plan): add a `VAL_MAP` check that returns `make_number(m->count)`.

**Registration** (in `register_builtins()`):
- Register `keys` (arity 1), `values` (arity 1), `has_key` (arity 2).

**Tests**:
- `keys({name: "alice", age: 30})` → `["name", "age"]`.
- `keys({})` → `[]`.
- `values({name: "alice", age: 30})` → `["alice", 30]`.
- `values({})` → `[]`.
- `has_key({a: 1}, "a")` → true.
- `has_key({a: 1}, "b")` → false.
- `has_key({a: nothing}, "a")` → true.
- `has_key({}, "a")` → false.
- `len({a: 1, b: 2})` → 2.
- `len({})` → 0.
- `keys(42)` → error "keys() requires a map".
- `has_key("str", "a")` → error "has_key() requires a map".

**Files touched**: `src/vm.c`, tests.
